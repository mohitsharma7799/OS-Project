ⓘ About This Simulator

Real-Time Scheduling Algorithm Simulator

This tool simulates and visualizes real-time CPU scheduling algorithms, helping users understand how tasks are prioritized, preempted, and scheduled under different policies. It is designed as an educational tool for Operating Systems courses.

Rate Monotonic (RM)

A fixed-priority preemptive algorithm. Tasks with shorter periods get higher priority. Optimal among fixed-priority algorithms for schedulable task sets. Priority is assigned at design time and does not change.

  • Priority = 1 / Period (shorter period → higher priority)
  • Preemptive: higher-priority tasks interrupt lower ones
  • May miss deadlines if CPU utilization exceeds the RM bound

Earliest Deadline First (EDF)

A dynamic-priority preemptive algorithm. The task with the nearest absolute deadline runs first. Optimal for uniprocessor systems — can achieve 100% CPU utilization without missing deadlines.

  • Priority changes dynamically based on current deadlines
  • Preemptive: a newly arrived task with an earlier deadline preempts
  • Theoretically optimal for single-processor scheduling

First-Come, First-Served (FCFS)

The simplest scheduling algorithm. Tasks are executed in the order they arrive. Non-preemptive — once a task starts, it runs to completion.

  • No priority mechanism — strictly arrival order
  • Simple but can cause long waiting times (convoy effect)
  • Not suitable for real-time systems with strict deadlines

Shortest Job First (SJF)

Selects the task with the smallest burst (execution) time. Non-preemptive — minimizes average waiting time for a given set of tasks.

  • Optimal for minimizing average waiting time
  • Requires knowing burst times in advance
  • Can cause starvation for long tasks

How to Use

  1. Add Tasks — Enter task parameters (ID, arrival, burst, deadline, period) and click "Add Task", or click "Load Sample" for a quick start.
  2. Select Algorithm — Choose RM, EDF, FCFS, or SJF from the dropdown.
  3. Run Simulation — Click the green "Run Simulation" button to execute.
  4. Analyze — View the Gantt chart, stats cards, simulation results table, and per-task analytics.
  5. Compare — Click "Compare All" to run all 4 algorithms and see a side-by-side comparison.
